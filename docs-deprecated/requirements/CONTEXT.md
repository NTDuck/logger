# Glossary

- **LogRecord**: A structured log event containing required fields (`Application_Name`, `Log_Level`, `Message`) and optional fields (`Timestamp`, `Trace_ID`, `Error_Code`). It is accepted as a JSON payload at the ingestion API.
- **Worker**: A background consumer that processes LogRecords from the Message Queue, performs optional enrichment (e.g. generating a Signature for alerts, extracting custom keys), and persists them to the Database.
- **AlertLock**: A Redis mechanism to deduplicate and threshold alerts. It uses a minute-granularity counter (`alert:count:{Signature}:{minute}`). If the count crosses the `AlertThreshold` (or 1 for `CRITICAL`), it attempts a `SET NX` on a fired lock (`alert:fired:{Signature}:{minute}`) to dispatch exactly one notification per window.
- **Signature**: The unique grouping identifier for an error, used in the AlertLock. It is derived either from the explicit `Error_Code` field or, if missing, by fuzzy-hashing the `Message` (stripping dynamic tokens like UUIDs, IPs, and digits).- **AccessScope**: The list of applications a specific user is authorized to view. Backed by PostgreSQL and cached in Redis as `rbac:user:<user_id>` for O(1) access by the WebSocket server.
- **AlertThreshold**: The configuration defining how many errors trigger an alert (e.g., `count` over `window_secs` per `Application_Name` and `Log_Level`). Backed by PostgreSQL, cached as a Redis Hash (`config:thresholds`), and held in-memory by Workers. Workers update their cache via a Redis Pub/Sub channel (`config:updated`) and fallback to polling the hash every 30s to guarantee consistency.- **AIWorker**: A dedicated, rate-limited consumer that processes `ERROR` and `CRITICAL` logs from a specialized queue (`logs:for_ai`). It performs lightweight asynchronous inference to classify errors without blocking the main ingestion pipeline.
- **AIInsight**: The output of the AIWorker's analysis, including classification, severity hints, and suggested fixes, stored in the `log_ai_insights` table and pushed to the UI for immediate context.
- **AppKey**: A unique authentication token (`X-API-Key`) required for applications to send logs to the Ingestion API. It maps to a specific `Application_Name` and is subject to token-bucket rate limits in Redis to ensure fair use and protect the global circuit breaker.
- **DeadLetterQueue**: A Redis Stream (`alerts:dead`) where alert payloads are permanently stored for administrative review if they fail to send after maximum exponential backoff retries. This ensures zero silent data loss for critical notifications.
